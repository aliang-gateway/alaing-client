## 注意事项

- [P] 任务 = 不同文件区域或只读操作，无编辑冲突
- [Story] 标签将任务映射到特定用户故事以实现可追溯性
- 每个用户故事应该独立可完成和可测试
- 在每个阶段或逻辑组后提交 Git
- 在任何检查点停止以独立验证故事
- 避免: 同时编辑同一文件的不同行导致冲突
- 建议: 使用浏览器的"实时重载"或"自动刷新"扩展以加快测试速度
- 重要: 所有修改都是样式层面，不影响后端逻辑或数据
