# 前后端接口差异分析与对其规划

## 文档概述

本文档基于以后端为主的策略，详细分析前端Admin Dashboard与后端Go API的接口差异，制定系统的对其方案和实施计划。

**项目基本信息**:
- 后端: Go语言智能网关服务 (127.0.0.1:56431)
- 前端: React + TypeScript Admin Dashboard
- WebSocket: 实时数据推送 (127.0.0.1:56433)

---

## 1. 核心差异分析

### 1.1 数据格式差异

#### **后端统一响应格式** (不可更改)
```json
{
  "code": 0,          // 0=成功, 非0=错误码
  "msg": "success",   // 响应消息
  "data": {}          // 实际数据
}
```

#### **前端当前数据结构** (需要适配)
- 前端组件直接使用Mock数据
- 没有考虑后端的统一响应格式
- 需要通过适配器层进行转换

### 1.2 现有API接口清单

#### **已实现的后端API**:
1. **Token管理** (2个接口)
   - `GET /token/get` - 获取当前Token
   - `POST /token/set` - 设置出站Token

2. **运行控制** (5个接口)
   - `GET /run/status` - 获取运行状态
   - `POST /run/start` - 启动服务
   - `POST /run/stop` - 停止服务
   - `POST /run/swift` - 切换模式
   - `POST /run/userInfo` - 设置用户信息

3. **代理管理** (8个接口)
   - `GET /proxy/current/get` - 获取当前代理
   - `POST /proxy/current/set` - 设置当前代理
   - `GET /proxy/registry/list` - 获取代理列表
   - `POST /proxy/registry/register` - 注册代理
   - `POST /proxy/registry/unregister` - 删除代理
   - `POST /proxy/registry/set-default` - 设置默认代理
   - `POST /proxy/registry/set-door` - 设置Door代理
   - `POST /proxy/registry/switch` - 切换代理

4. **配置管理** (2个接口)
   - `GET /config/get` - 获取配置
   - `GET /config/list` - 获取配置列表

5. **日志管理** (5个接口 + 1个WebSocket)
   - `GET /api/logs` - 获取日志
   - `POST /api/logs/clear` - 清空日志
   - `POST /api/logs/level` - 设置日志级别
   - `GET /api/logs/config` - 获取日志配置
   - `POST /api/logs/config` - 更新日志配置
   - `WebSocket /api/logs/stream` - 实时日志流

6. **通用WebSocket**
   - `WebSocket /ws` - 通用连接 (端口56433)

## 3. 对齐方案总体规划

1. **后端为主**: 保持后端API的稳定性和一致性
2. **前端适配**: 通过适配器层解决数据格式差异
3. **渐进式**: 分阶段实施，先解决高匹配度功能
4. **可扩展**: 为未来功能扩展预留接口

## 5. 前端适配层设计

### 5.1 数据适配器架构

```typescript
// src/utils/adapters.ts
export class DataAdapters {
  // 系统状态适配器
  static systemStatus(backend: BackendSystemStatus): SystemStatus {
    return {
      running: backend.tun_running,
      mode: backend.current_mode,
      uptime: this.calculateUptime(backend)
    };
  }

  // 日志条目适配器
  static logEntry(backend: BackendLogEntry, index: number): LogEntry {
    return {
      id: index,
      timestamp: this.formatTimestamp(backend.timestamp),
      level: backend.level,
      message: backend.message,
      source: backend.source
    };
  }

  // 代理节点适配器
  static proxyNode(backend: BackendProxyInfo, index: number): ProxyNode {
    return {
      id: `proxy-${index}`,
      name: backend.name,
      location: this.extractLocation(backend.config),
      status: 'unknown', // 需要额外API获取
      latency: 0,
      load: 0
    };
  }
}
```

### 5.2 API服务层设计

```typescript
// src/services/
export class SystemService {
  async getStatus(): Promise<SystemStatus> {
    const backend = await api.get<BackendSystemStatus>('/run/status');
    return DataAdapters.systemStatus(backend);
  }

  async getUserInfo(): Promise<UserInfo> {
    const backend = await api.get<BackendUserInfo>('/user/info');
    return DataAdapters.userInfo(backend);
  }
}

export class LogService {
  async getLogs(level?: string): Promise<LogEntry[]> {
    const backend = await api.get<BackendLogResponse>('/api/logs', { level });
    return backend.entries.map((entry, index) =>
      DataAdapters.logEntry(entry, index)
    );
  }
}
```

### 5.3 React Hooks封装

```typescript
// src/hooks/
export function useSystemStatus() {
  const [data, setData] = useState<SystemStatus | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  const fetchData = async () => {
    try {
      setLoading(true);
      const result = await new SystemService().getStatus();
      setData(result);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };

  useEffect(() => {
    fetchData();
    const interval = setInterval(fetchData, 5000);
    return () => clearInterval(interval);
  }, []);

  return { data, loading, error, refetch: fetchData };
}
```

func corsMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        w.Header().Set("Access-Control-Allow-Origin", "http://localhost:3000")
        w.Header().Set("Access-Control-Allow-Methods", "GET, POST, PUT, DELETE, OPTIONS")
        w.Header().Set("Access-Control-Allow-Headers", "Content-Type, Authorization")

        }

## 8. 风险评估和解决方案
