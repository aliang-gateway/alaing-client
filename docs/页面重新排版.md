# 页面重新排版

1. 默认首页，主要展示状态等信息
  1. 右上角展示用户的状态信息，我希望他是一个小的方形的方块；
     1. 头像（固定一个默认的头像）
     2. 状态：是否开启，当前服务的运行状态；
     3. 套餐：
        1. 套餐名字
        2. 套餐的日期
  2. 页面左右排版
     1. 左边展示一个开关按键，正在运行后出现一个波浪的特效，让按钮动起来；
     2. 下方出现一个卡片，展示当前运行的状态
        1. 运行模式：http，tun两种，可以选择和切换；
        2. 是否设置流量转发，展示对应的转发协议：
           1. socks5的转发信息
           2. 或者http的转发信息
           3. direct
        3. 证书状态，展示证书是否安装
           1. 详细按钮，点击后进入证书详细
           2. 安装证书：点击后执行安装任务
     3. 增加一些功能按钮
        1. 快速设置，一个工具按钮，点击后弹出一个菜单：
           1. 一键配置opencode
           2. 一键配置claude code
           3. 一键配置codex
           4. 一键生成openclaw配置
           5. 提交其他支持
        2. 快速聊天
           1. 点击后弹出出一个聊天框；
              1. 展示聊天的数据，可以在里边对话
              2. 保存的历史对话数据
              3. 提示所有数据都在本地
           2. 检索历史的聊天数据
        3. 更多设置按钮，点击后进入更多设置页面
        4. 最下方展示用户的套餐情况，比如剩余量等
     4. 右侧展示检测出来的流量信息：
        1. 上下布局
           1. 上边左右布局：
              1. 左边是一个饼状图，展示各个域名的流量占比
              2. 右侧展示流量图，主流域名的流量，最小单位是15s
              3. 流量的内容，只检查部分流量，支持配置：
                 1. cursor
                 2. openai
                 3. claude
                 4. chatgpt
                 5. github的coplit 
           2. 下边模块，展示最近请求的每一条记录：
              1. 展示各个api的访问记录，是一条一条的，点击后可以展开；
              2. 最多50条，默认顶部保持最新用户可以滑动；
     5. 整个页面的在一个屏幕内

2. 设置页面这是一个新的页面，里边有全量的设置

   1. 顶部有一个导航栏，水平放置，菜单内容：**规则设置、用户信息、日志监控、系统设置**； 最右上角展示（用户头像、运行状态的卡片）

   2. 规则设置，是一个大型的编辑框，用来表明哪些流量属于什么类型，一共支持：claude，cursor，copliot，Gemini，grok，openai，DeepSeek，glm，kimi等等，有助于劫持特定域名的流量

      1. cursor类型的流量域名
      2. openai的类型等等，
      3. copliot对应的域名等等
      4. ……

   3. 用户信息：

      1. 展示用户的信息：账号、用户名，api-key列表
      2. api-key展示风格是一行一行的，以及这个api-key对应的api接口类型，比如claude风格，openai风格，gemini风格等；
      3. 用户的套餐信息，可以是无
         1. 分别有pro，plus，utrl，free
      4. 用户的余额
      5. 用户的套餐使用情况
      6. 配置管理
         1. 用户的常见的配置文件，支持：
            1. opencode.json；同步到云端
            2. Oh-my-opencode.json的配置，同步到云端
            3. claude的环境变量设置的命令
            4. codex的配置文件；
            5. vibe-kanban

   4. 日志监控

      1. 滚动展示当前系统的日志，用WebSocket来实现
      2. 可以设置日志的level，在前端来过滤日志；也可以设置系统后端的日志等级；
      3. 支持开启关闭、刷新；并且展示日志状态

   5. 系统设置

      1. 系统运行状态

         1. 运行模式：tun或者http
         2. 运行状态展示
         3. 模式切换

      2. 软件状态

         1. 是否注册到系统(MACOS)
         2. 如果用户选择注册到系统，则在macos的/Library/LaunchDaemons下注册；

      3. 证书状态

         1. 卡片形式展示证书状态：

         2. 展示系统是否已经安装证书，是否已经信任证书，是否已经导出；

         3. 展示证书的信息，比如主题、办法这、有效期、指纹等

         4. 证书操作：

            1. 检查安装状态；
            2. 导出证书
            3. 下载证书；
            4. 安装证书；
            5. 移除证书
